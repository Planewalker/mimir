Zimrazim wrote:
Now, do any of you fine players have a suggestion about the best way to display a map grid/combat grid in a PbP game like this? I suppose there's the "draw the map by hand, scan it in, and make a gif/jpg file out of it" method, but that seems like a rather primitive way to go about things these days.
In a game I'm running on another site, I made up a small grid in photoshop, with different-colored tokens to mark each player, and uploaded a new image via photobucket for each round of combat. If you have any software of that sort, that's a pretty decent way to go about it, I find. That way you don't have to redraw a new map every round, you'd already have one ready to go every time you need it, with markers for the characters. All you'd really need to worry about is any relevant terrain and enemy markers per encounter, and moving the placeholders each round (which is especially easy if you use layers). I could send you a link if you'd like to see mine.
