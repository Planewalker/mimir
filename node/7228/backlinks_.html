I *think* I have diagnosed the autologin issues. Here's what I've found:

- First visit to Planewalker: Obviously not logged in, first visit. Log in, check autologin, get a cookie (named planewalker, set to work on any .planewalker.com site) that's set to expire in a month's time. No problems here.

- Second visit to Planewalker: The user has an autologin cookie, so the system reads it and logs the user in. This process apparently destroys the cookie, without replacing it - at least, at the time I write this, I have no cookie named planewalker.

- Third visit to Planewalker: The autologin cookie was destroyed by the previous autologin process, so there's no current data and no autologin occurs. Seems to match the behavior reported by Nick and other forumgoers.

Behavior's occuring on a Mozilla 1.7.5 browser, BTW. Hope this proves helpful.
